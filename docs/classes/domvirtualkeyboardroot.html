Class DOMVirtualKeyboardRoot

A DOMRoot with similar functionality to VirtualKeyboardRoot. In this version updateVisibility doesn't exist. Instead, just call update like in DOMRoot.

constructor

  • new DOMVirtualKeyboardRoot(keyboardDriver: KeyboardDriver, keyboardTemplate?: VirtualKeyboardTemplate, theme?: Theme): DOMVirtualKeyboardRoot

  • Overrides DOMRoot.constructor

    • Defined in core/DOMVirtualKeyboardRoot.ts:18

    Creates a new VirtualKeyboardRoot.

    Sets child to a new Margin containing a VirtualKeyboard with the given keyboard and keyboard template and child's inherited theme. Also sets up a pointerStyleHandler which simply sets the CSS cursor style of domElem. Creates domElem and domCanvasContext.

    Parameters

    • keyboardDriver: KeyboardDriver
    • keyboardTemplate: VirtualKeyboardTemplate = ...

      By default, the virtual keyboard template is defaultVirtualKeyboardTemplate

    • theme: Theme = ...

      If none supplied, then the default theme found in Theme.constructor is used

    Returns DOMVirtualKeyboardRoot

    Dispatches an Event to this root's child by calling Widget.dispatchEvent. Updates foci capturers and notifies drivers by calling Driver.onFocusCapturerChanged if the capturer changes. Does nothing if root is disabled.

    Note that if an event with a focus is dispatched and no widget captures the event due to the widget not existing anymore or being disabled, the focus type of the event will be cleared in the root with clearFocus.

    Parameters

    • event: Event

    Returns boolean

    Returns true if the event was captured

  • Overrides DOMRoot.update

    • Defined in core/DOMVirtualKeyboardRoot.ts:55

    Update DOMRoot.

    If root is disabled, domElem's display style is set to 'none', hiding it.

    Calls preLayoutUpdate, resolveLayout, postLayoutUpdate and paint.

    Also updates the visibility of this root; if the keyboard driver has no focused root, then the root is disabled, else, it is enabled.
